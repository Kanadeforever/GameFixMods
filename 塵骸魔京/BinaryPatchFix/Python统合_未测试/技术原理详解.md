# 技术原理详解

## 鼠标冻结修复

### 根因分析

Windows 10 (21H2+) 在特定的 IME（输入法编辑器）交互场景下，`ImmGetDefaultIMEWnd` / `ImmGetContext` / `ImmSetOpenStatus` 系列 API 与 `AttachThreadInput` 的组合调用会导致 **全局鼠标冻结**。

游戏在每一帧或特定 UI 切换时调用这些 API（常见于日文游戏使用古老 DirectInput 或 Win32 IME 接口），在 Win10 上触发了一个内核态的死锁路径：线程附加后 IME 窗口消息被错误路由到已退出的 worker 线程，导致鼠标消息队列永久阻塞。

### 修复方案

采用 **API Hook** 方式，将 4 个有问题的 API 替换为空操作：

| API | Hook 函数 | 返回值 |
|:----|:----------|:-------|
| `ImmGetDefaultIMEWnd` | `Hk1` | `NULL` |
| `ImmGetContext` | `Hk2` | `NULL` |
| `ImmSetOpenStatus` | `Hk3` | `FALSE` |
| `AttachThreadInput` | `Hk4` | `FALSE` |

### 加载方式

通过修改游戏 PE 文件的导入表（Import Table），新增两条导入记录：

1. `imm32.dll!ImmGetContext` — 确保 `imm32.dll` 被加载
2. `fix.dll!FixInit` — 加载 fix.dll，触发 `DllMain` 执行 Hook

Hook 时机：`DllMain` (DLL_PROCESS_ATTACH) → `GetModuleHandle` 定位已加载的模块地址 → `GetProcAddress` 找到函数入口 → 5 字节 JMP 改写到空函数。

### 为什么是 imm32.dll + fix.dll 双导入？

- `imm32.dll` 必须被显式导入，否则 `GetModuleHandleA("imm32.dll")` 返回 NULL
- `fix.dll` 通过导入表加载自动执行 `DllMain`，不需要游戏代码调用任何函数

## 注册表弹窗绕过

### 根因分析

游戏在启动时读取注册表 `HKLM\SOFTWARE\NitroPlus\塵骸魔京` 中的 `serial` 值。如果该值不存在（Win10 默认无此注册表项），游戏弹出模态序列号输入对话框。64 位系统上，注册表重定向使问题更复杂。

### 修复方案

直接修改 `system.dll` 中的一条条件跳转指令：

```
0x168C6:  test eax, eax         ; 序列号验证结果
0x168C8:  jne 0x169C0           ; 成功 → 跳过弹窗
         ↓
0x168C8:  jmp 0x169C0           ; 强制跳过弹窗（无论验证结果）
```

### 替代方案

也可以通过提供注册表 `.reg` 文件解决（见 `archive/`），但二进制补丁方式对用户最友好——零配置，复制即用。

## 兼容性说明

详细兼容性报告见 `docs/compatibility_report.md`。

| 风险项目 | 严重度 | 说明 |
|:---------|:-------|:------|
| `IsBadReadPtr/WritePtr/CodePtr` | 低 | Win10 已是空操作 |
| `GetVersionExA` | 低 | 返回虚假版本号，不影响运行 |
| `PulseEvent` | 低 | 罕见线程唤醒遗漏 |
| `DirectDrawCreateEx` | 低 | Win10 通过 D3D9 模拟 |
| `SetCursorPos` | 低 | UIPI 不影响同进程 |

## 潜在隐患（非危急，源于原始游戏）

| API | 风险 | 说明 |
|:----|:-----|:------|
| `IsBadReadPtr/WritePtr/CodePtr` | 🔵 低 | Win10 上已是空操作，不检查内存。游戏不应依赖它们 |
| `GetVersionExA` | 🔵 低 | Win10 返回虚假版本号(6.2)，游戏可能误判为 Win8。实测不影响运行 |
| `PulseEvent` | 🔵 低 | Win10 上不可靠，可能导致罕见线程唤醒遗漏 |
| `DirectDrawCreateEx` | 🔵 低 | Win10 通过 D3D9 模拟 DirectDraw，2D 游戏一般正常 |
| `SetCursorPos` | 🔵 低 | UIPI 可能阻止跨完整性级别光标移动，同进程无影响 |

## 结论

无其他高危兼容性隐患。五个低风险项都是原始游戏固有特征，不属于新系统特有问题，且实测不影响正常运行。
